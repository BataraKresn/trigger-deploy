üîó API Endpoints

POST /api/auth/login
Authenticate user with username and password to access protected resources.
Parameters
username string
User's username (any username accepted)
password string
Password (check LOGIN_PASSWORD in .env file)
Example Request
curl -X POST http://localhost:5000/api/auth/login \ -H "Content-Type: application/json" \ -d '{ "username": "admin", "password": "admin123" }'
GET /api/servers
Retrieve list of all configured servers with their current status and information.
Example Request
curl -X GET http://localhost:5000/api/servers \ -H "Authorization: Bearer YOUR_SESSION_TOKEN"
Response Example
[ { "alias": "web-server-01", "ip": "192.168.1.100", "name": "Production Web Server", "user": "deploy", "script_path": "/opt/deploy/deploy.sh", "status": "online" } ]
POST /api/deploy
Trigger deployment to a specific server using server alias and deploy token.
Parameters
server_alias string
Target server alias from servers.json
deploy_token string
Deploy token (check DEPLOY_TOKEN in .env)
Example Request
curl -X POST http://localhost:5000/api/deploy \ -H "Content-Type: application/json" \ -d '{ "server_alias": "web-server-01", "deploy_token": "SATindonesia2026" }'
GET /api/health/{target}
Perform health check on specified target domain or IP address.
Path Parameters
target string
Domain or IP to check (e.g., google.co.id)
Example Request
curl -X GET http://localhost:5000/api/health/google.co.id
GET /api/logs
Retrieve list of available log files with metadata.
Example Request
curl -X GET http://localhost:5000/api/logs
INFO Authentication Methods
This API supports two authentication methods:
Session-based Authentication
Cookie: session automatic
Automatically handled after login via web interface
Token-based Authentication
X-Deploy-Token header
Deploy token for specific operations (from .env DEPLOY_TOKEN)
Configuration
# Check your .env file for these values: LOGIN_PASSWORD=admin123 DEPLOY_TOKEN=SATindonesia2026 # Or check defaults in code if .env not found
